Возросшее количество (+5) страховых компаний приведет к замедлению работы API ins-product-aggregator, тк все запросы к партнерам для получения данных о доступных продуктах выполняются inline и все ошибки транслируются в ответы API ins-product-aggregator.
Из-за того, что запросы к ins-product-aggregator запускаются по расписанию и требуют немедленного ответа, при ошибках, не зависящих от ins-product-aggregator расписание будет нарушено и требуется вносить изменение в логику потребителей по повторению сбойных запросов полностью, без выделения неработающих сервисов партнеров. Информация от работающих партнеров окажется недоступной

Использование event-streaming считаю нецелесообразным в силу неконсистентной природы данных о продуктах партнеров. Агрегатор получает данные регулярными запросами к партнерам. Эти данные всегда представляют из себя полный список продуктов без информации об изменениях относительно предыдущих запросов и не могут использоваться в качестве надежной основы для организации event-streaming-а
Количество партнеров и их продуктов выглядит управляемым и находится в пределах, не способных вызвать избыточную передачу данных.
